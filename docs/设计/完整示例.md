# 完整示例

当前完整示例在：

```text
flow-kernel-example/src/test/java/io/github/openflowkernel/example/ProcessTaskFormFlowTest.java
```

它是测试，不是生产应用。目的很明确：不依赖 Camunda、MySQL、真实表单平台，也能跑通并理解核心设计。

## 示例覆盖什么

当前示例覆盖：

- 启动流程。
- 引擎触发 task-created。
- 创建业务任务实例。
- 通过事件执行任务初始化。
- `FormTask` 创建表单。
- `FormService` 提交表单。
- 发布 `FormSubmitted`。
- 完成 `TaskRelation`。
- 完成业务任务。
- 推进引擎。
- 完成业务流程。
- 初始化 packet。
- application 任务提交 packet value。
- 流程完成事件更新 packet 状态。
- 任务 candidate diff update，并按 `ucid` 去重。
- 重复表单提交幂等保护。
- 配置化表单 input/output 映射。
- 多表单 `FORM_DEF_MAP` / `FORM_INPUT_MAP` / `FORM_OUTPUT_MAP`。
- 任务重入。
- 单子流程。

## 主流程

```text
application-review
-> application FormTask
-> review FormTask
-> process end
```

身份映射和未来 Camunda 适配器一致：

```text
engine businessKey = processInstanceId
engine task code = taskCode
```

## 示例装配了什么

测试 fixture 中装配：

- `ProcessDefinitionRepository`
- `ProcessInstanceRepository`
- `TaskInstanceRepository`
- `TaskRelationRepository`
- `InMemoryWorkflowEngine`
- `InMemoryFormService`

## Camunda 闭环示例

真实 Camunda adapter 的完整闭环在：

```text
flow-kernel-engine-camunda/src/test/java/io/github/openflowkernel/engine/camunda/CamundaProcessTaskFormIntegrationTest.java
```

它使用 Camunda in-memory engine 部署两节点 BPMN，并让完整链路通过：

```text
ProcessService.start
-> CamundaWorkflowEngine.start
-> Camunda task-created callback
-> WorkflowEventListener 创建业务 TaskInstance
-> FormTask 创建表单和 TaskRelation
-> FormService.submit
-> FormSubmittedEventListener 完成 relation
-> TaskInstance.complete
-> CamundaWorkflowEngine.completeTask
-> Camunda process end callback
-> ProcessService.complete
```

## 阅读顺序

建议按这个顺序读示例：

1. 流程定义注册。
2. 任务处理器注册。
3. `ProcessService.start`。
4. `WorkflowEventListener.onTaskCreated`。
5. `TaskInitializationEventListener`。
6. `FormTask.init`。
7. `FormService.submit`。
8. `FormSubmittedEventListener`。
9. `TaskRelationService.complete`。
10. `TaskInstance.complete`。
11. `WorkflowEngine.completeTask`。
12. `WorkflowEventListener.onTaskCompleted`。
13. `TaskInstance.postComplete`。

这条链路就是当前设计核心。

## 状态、数据、事件要点

- Process 状态从 `INIT` 到 `NORMAL_END`。
- Task 状态从 `CREATE` 到 `INIT`，表单提交后到 `COMPLETE`。
- Form 状态从 `ACTIVE` 到 `SUBMITTED`。
- TaskRelation 在表单提交后完成，所有 relation 完成后触发任务完成。
- Packet 在流程启动后初始化，任务完成后 commit value，流程完成事件后变为 `COMPLETED`。
- Candidate 通过 `TaskInstance.init()` 读取 `CANDIDATE_SELECTION`，调用
  `TaskCandidateService.refresh` 和 `TaskCandidateSource` 初始化候选人；同时保留
  `diffAndUpdateCandidates` 的去重、增量新增、软删除旧候选人语义。
- Event 负责把引擎回调、任务初始化、表单提交、流程完成串起来。
- Multi-form 会创建多个 `FormInstance` 和多个 `TaskRelation`，只有所有表单 relation 完成后任务才完成。
- Multi-subprocess 会创建多个 child `ProcessInstance` 和多个 `subProcessTaskType` relation，只有所有子流程完成后父任务才完成。
