# 快速开始

本文给第一次打开仓库的读者一条最短路径：先跑通，再读设计。

## 环境

- JDK 17
- Gradle wrapper：使用仓库内 `./gradlew`

默认路径不需要 Camunda 服务端、MySQL 或真实表单平台。

## 一条命令跑通

```bash
./gradlew test --no-daemon --warning-mode all
```

这条命令会跑通：

- in-memory integrated example
- JDBC repository tests
- Camunda in-memory adapter tests
- Process-Task-Form lifecycle
- Event, Packet, Candidate, Form, SubProcess baseline

## 第一条完整链路

先看：

```text
flow-kernel-example/src/test/java/io/github/openflowkernel/example/ProcessTaskFormFlowTest.java
```

重点方法：

```text
completesProcessThroughTwoFormTasks
```

它展示：

```text
ProcessService.start
-> engine task-created
-> TaskInstance.init
-> FormTask creates Form + TaskRelation
-> FormService.submit
-> FormSubmitted event
-> TaskRelation complete
-> TaskInstance.complete
-> WorkflowEngine.completeTask
-> process complete
```

这条链路不依赖 Camunda 或 MySQL，方便先理解模型。

## Camunda 闭环

再看真实 Camunda adapter 测试：

```text
flow-kernel-engine-camunda/src/test/java/io/github/openflowkernel/engine/camunda/CamundaProcessTaskFormIntegrationTest.java
```

重点方法：

```text
processTaskFormFlowCompletesThroughRealCamundaEngine
```

它使用 Camunda in-memory engine，证明：

```text
Camunda businessKey = kernel ProcessInstance.id
Camunda taskDefinitionKey = kernel TaskDefinition.code
```

Camunda 负责路由，业务流程事实仍由 kernel 持有。

## JDBC 和 DB

默认测试使用 H2 MySQL mode。建表语句在：

```text
db/schema/h2/open-flow-kernel-jdbc-schema.sql
db/schema/mysql/open-flow-kernel-jdbc-schema.sql
```

JDBC schema 字段保护测试：

```text
flow-kernel-persistence-jdbc/src/test/java/io/github/openflowkernel/jdbc/persistence/JdbcSchemaParityTest.java
```

MySQL/Testcontainers 测试需要 Docker：

```bash
./gradlew :flow-kernel-persistence-jdbc:mysqlIntegrationTest --no-daemon --warning-mode all
```

没有 Docker 时，可以先编译 MySQL 集成测试：

```bash
./gradlew :flow-kernel-persistence-jdbc:mysqlIntegrationTestClasses --no-daemon --warning-mode all
```
