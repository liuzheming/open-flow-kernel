# 发布前检查清单

本文记录 `open-flow-kernel` 从当前抽取 baseline 走向公开 GitHub 仓库前必须确认的事项。

## 当前结论

当前仓库已经具备第二轮 / 第三轮核心审计证据。MySQL/Testcontainers runtime 不作为核心方案公开的阻塞项，但仓库还不是可正式发布的开源 artifact。

原因：

- `LICENSE` 仍处于法务确认状态。
- `SECURITY.md` 里的安全联系人仍是占位描述，需要发布前配置正式渠道。

## 必须完成

| 项 | 当前状态 | 发布前动作 |
| --- | --- | --- |
| 许可证 | `LICENSE` 明确仍在法务确认中 | 替换为批准的开源许可证 |
| 安全联系人 | `SECURITY.md` 已有范围说明，但没有正式公开联系人 | 配置正式安全披露渠道 |
| CI | `.github/workflows/ci.yml` 运行 clean build 并编译 MySQL 集成测试 | 首次推送 GitHub 后确认 CI 真实通过 |
| clean build | `./gradlew clean build --no-daemon --warning-mode all` 本地通过 | 发布前复跑 |
| 默认测试 | `./gradlew test --no-daemon --warning-mode all` 本地通过 | 发布前复跑 |
| MySQL runtime | `mysqlIntegrationTestClasses` 可编译；runtime 受 Docker 限制未证明 | 可选外部验证项；Docker 可用后运行 `mysqlIntegrationTest` |
| 公开风险扫描 | 已清理本机路径、内部域名、公司包名、内部系统缩写 | 发布前复扫关键词 |
| DDL parity | H2/MySQL DDL 已有文档和 `JdbcSchemaParityTest` 保护 | schema 改动时同步更新 |
| API 边界 | `docs/API与边界/模块边界.md` 和 `docs/API与边界/公共API边界.md` 已说明 public 分层 | API 改动时同步更新 |

## 发布前命令

```bash
./gradlew clean build --no-daemon --warning-mode all
./gradlew test --no-daemon --warning-mode all
./gradlew :flow-kernel-persistence-jdbc:mysqlIntegrationTestClasses --no-daemon --warning-mode all
```

确认 JDBC jar 内包含 schema resource：

```bash
jar tf flow-kernel-persistence-jdbc/build/libs/flow-kernel-persistence-jdbc-0.1.0-SNAPSHOT.jar \
  | rg '^open-flow-kernel-jdbc-schema.sql$'
```

Docker 可用时再运行：

```bash
./gradlew :flow-kernel-persistence-jdbc:mysqlIntegrationTest --no-daemon --warning-mode all
```

该命令用于证明真实 MySQL 兼容性，不作为当前核心复刻闭环的默认验收门槛。

公开风险关键词复扫：

```bash
# 将 <sensitive-keyword-regex> 替换成维护中的私有风险关键词正则。
rg -n "<sensitive-keyword-regex>" . -g '!**/build/**' -g '!**/.gradle/**' -g '!**/.git/**'
```
