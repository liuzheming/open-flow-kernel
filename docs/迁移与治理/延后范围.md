# 延后范围

本文集中记录当前不属于 第三轮 有界核心复刻的能力。

这些能力不是被否定，而是不能混进当前 extraction baseline。后续如果要做，应该单独定义目标、参考来源、测试和迁移边界。

## 生产化包装

当前不包含：

- production Spring Boot starter
- auto-configuration
- deployment management
- history query facade
- multi-tenant Camunda deployment policy
- Maven Central release execution
- full release automation

原因：第三轮 的目标是复刻核心设计，不是交付完整生产平台。

## 业务平台能力

当前不包含：

- UI / admin console
- approval-platform full behavior
- action orchestration platform
- audit task system
- task retry UI
- reject / rollback product workflow
- process graph rendering
- internal personnel/auth/signing/RPC implementations
- internal permissions, notification, resource-locking or observability integrations

原因：这些是业务平台或内部基础设施，不是 `Process -> Task -> Form` 内核本身。

## Candidate 后续能力

当前已完成：

- `TaskCandidateSource` SPI baseline
- `TaskInstance.initCandidate`
- candidate diff update
- `process_task_inst_candidate`
- `candidate_inst_relation` JDBC repository baseline

延后：

- Groovy `CANDIDATE_SELECTION` script engine
- full role/personnel/auth source resolution
- assignment UI
- missing-candidate notification side effects
- search/todo sync event side effects
- complex todo-node query

## Form 后续能力

当前已完成：

- single-form baseline
- generic multi-form maps：`FORM_DEF_MAP`, `FORM_INPUT_MAP`, `FORM_OUTPUT_MAP`
- relation wait and submit completion

延后：

- menu/detail form rendering
- old/new form relation side effects
- form resource cleanup hooks
- product-specific form buttons and paths

## SubProcess 后续能力

当前已完成：

- single subprocess
- generic `SUB_PROCESS_DEF_CODE_MAP`
- subprocess relation wait
- single-child `FORM_OUTPUT`

延后：

- business-specific subprocess branches
- candidate propagation
- packet init propagation
- multi-child output aggregation

## Packet 后续能力

当前已完成：

- pointer/value/history
- commit CAS
- expire reset
- typed `Packet<V>`
- `PacketFactory`
- `PacketInitManager`
- process-start packet init hook

延后：

- business packet subclasses
- contract/signing/document helper methods
- parent/root packet commit helpers
- operational no-reset override
- field-level packet query/index tables

## Event 后续能力

当前已完成：

- EventV2 baseline
- event-first record
- after-commit boundary abstraction
- per-listener delivery state
- retry/backoff/permanent failure
- replay
- JDBC event record/delivery persistence

延后：

- Spring transaction adapter
- production async executor
- scheduler integration
- operations API for failed events
- exact original JOOQ record class parity

## 持久化和数据库

当前已完成：

- H2/MySQL DDL files
- JDBC repository baseline
- DDL parity documentation
- `JdbcSchemaParityTest`
- MySQL/Testcontainers source set and compile task

延后 / 未验证：

- real MySQL runtime compatibility until Docker is available
- exact original table/index naming parity where current adapter intentionally differs
- multi-database support beyond the current JDBC baseline
- full JOOQ/generated-record parity

## 可靠性改进

这些能力可能有工程价值，但原参考核心没有以当前形式建模，不能在复刻阶段直接引入：

- durable command store
- outbox as a replacement for the reference event model
- reconciliation worker
- new `COMPLETING` task status
- new process/task state transitions absent from reference enums
- alternative transaction model that changes task-complete-before-engine-complete ordering

这些属于复刻之后的改进讨论，不能改变当前设计精髓。
