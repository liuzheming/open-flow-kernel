# 文档导航

这组文档的目标是让读者快速理解 `open-flow-kernel` 的设计取舍：

- 业务流程内核使用 Process、Task、Form 三层模型承载业务语义。
- Camunda 只作为 BPMN 状态机，不侵入核心领域模型。
- Event、Packet、Candidate、SubProcess、Persistence 都是围绕这个核心模型展开的 adapter 或扩展点。

如果只读最少内容，按这个顺序：

1. [快速开始](快速开始/快速开始.md)：拉起项目、跑测试、知道当前边界。
2. [设计精髓](设计/设计精髓.md)：理解为什么是 Process-Task-Form + Camunda。
3. [完整示例](设计/完整示例.md)：从一条端到端流程看核心对象怎么协作。
4. [Camunda 作为状态机](设计/Camunda作为状态机.md)：理解 Camunda 与业务内核的边界。
5. [核心接口](API与边界/核心接口.md)：看可复用 API/SPI。
6. [迁移指南](迁移与治理/迁移指南.md)：把这套方案映射到老流程系统治理。
