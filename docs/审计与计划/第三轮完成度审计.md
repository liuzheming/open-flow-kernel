# 第三轮完成度审计

本文按 [项目计划](项目计划.md) 的第二轮 / 第三轮标准，对当前仓库做完成度审计。

结论：

- 第二轮的 Camunda + Process-Task-Form 闭环、示例和核心文档已经完成。
- 第三轮的有界核心对齐、开源交付文档、DDL 和测试保护已经按当前环境最大可验证范围完成。

- 当前不能宣称正式开源发布的原因是 `LICENSE` 仍处于法务确认状态。

## 验收命令

默认验收：

```bash
./gradlew test --no-daemon --warning-mode all
```

MySQL/Testcontainers 编译保护：

```bash
./gradlew :flow-kernel-persistence-jdbc:mysqlIntegrationTestClasses --no-daemon --warning-mode all
```

真实 MySQL runtime 验证命令：

```bash
./gradlew :flow-kernel-persistence-jdbc:mysqlIntegrationTest --no-daemon --warning-mode all
```

## 未证明项

MySQL/Testcontainers runtime 尚未完成真实运行验证，但不作为当前核心复刻闭环阻塞项。

当前原因：

```text
Docker daemon 不可连接。
```

这不影响默认验收命令，也不影响 H2/MySQL mode 下的 JDBC repository 行为验证。当前结论应表述为：MySQL DDL 和 Testcontainers 测试入口已提供，真实 MySQL compatibility 需要 Docker 可用后再证明。

正式开源发布还需要替换 `LICENSE` 中的等待法务确认文案，并配置 `SECURITY.md`
中的公开安全联系人。发布前检查清单见 [发布前检查清单](../迁移与治理/发布前检查清单.md)。

## 明确延后

集中边界见 [延后范围](../迁移与治理/延后范围.md)。摘要如下：

- 生产级 Spring Boot starter
- Maven Central 发布执行

- 完整可观测性体系
- benchmark
- 内部人员、权限、签约、RPC 实现
- 完整审批平台行为
- 参考核心不存在的新流程概念
- durable command store / reconciliation worker 等复刻之后的可靠性改进

这些能力后续可以单独立项，但不应改变当前从原设计抽取出的核心模型。
