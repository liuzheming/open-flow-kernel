# 高保真审计

本文用于审计当前实现是否仍然忠于参考设计。

当前阶段是抽取和复刻，不是重新设计。参考实现的生命周期、状态语义、事件方向、配置键和持久化含义优先于“看起来更现代”的重构。

## 审计规则

1. 实现组件前先读参考类和直接依赖。
2. 保留公共概念、生命周期顺序、状态语义、事件顺序、配置键、持久化含义和扩展点。
3. 内部基础设施只替换成最薄 adapter。
4. 参考实现的已知问题先记录，不在复刻阶段擅自修正。
5. 不新增参考实现没有的状态、命令、事件或事务模型。
6. 已迁移组件必须能在 [设计来源与偏离](设计来源与偏离.md) 中找到参考来源。

## 冻结的重新设计

复刻阶段不要引入：

- `COMPLETING` task status。
- engine command store。
- 用 outbox 替代参考 event system。
- reconciliation worker。
- 强制性的 correlation/causation kernel contract。
- 参考枚举中不存在的新流程/任务状态。
- 参考事件模块之外的新 retry 语义。
- 替代参考表语义的新表模型。

这些可以在高保真 baseline 完成后作为后续改进讨论。

## 当前 JDBC baseline

已实现：

- `flow-kernel-persistence-jdbc`
- schema initializer
- process definition / process instance / task instance / task relation repositories
- process/task guarded status update
- unique request id
- subprocess relation fields
- packet pointer/value repositories
- task candidate repositories
- event record/delivery store
- bounded parity repositories：`candidate_inst_relation`、`process_inst_relation`、`process_log`、`process_inst_data_init_config`
- H2 MySQL mode repository tests
- MySQL/Testcontainers source set 编译保护

仍属后续或外部验证：

- 真实 MySQL runtime compatibility，需要 Docker 可用后运行。
- Spring transaction adapter 和 scheduler。
- 原定义缓存行为。
- 平台专属候选人来源和副作用。

## 后续改进记录

这些是参考实现或当前抽取的已知风险，但不在复刻阶段擅自改造：

- 业务任务状态更新和引擎完成之间的事务假设。
- 容器查找和全局依赖。
- 环境分支。
- 反射和可变事件注册。
- 大 Map 和隐式序列化。
- 分布式锁依赖。

改进设计必须在当前验收套件稳定通过后单独立项。
